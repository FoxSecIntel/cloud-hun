# Contributing to Cloud-Hun

Thanks for your interest in contributing to **Cloud-Hun** - a lightweight cloud exposure and detection resource focused on AWS, Azure and GCP.

## How to Contribute

### 1. Issues & Feature Requests
If you find a gap in the detection content, misconfiguration guidance or examples, please open an issue with:
- A clear title  
- Cloud provider affected  
- Minimal reproduction steps (if applicable)

### 2. Adding New Detection Content
For new rules, queries or artefacts, please include:
- Cloud provider (AWS / Azure / GCP)
- Use case or threat scenario
- Detection logic (query, policy or script)
- References / documentation (if available)

### 3. Folder Structure
Please place contributions in the correct directory:
- `aws/` for AWS detections
- `azure/` for Azure
- `gcp/` for GCP
- `examples/` for sample scripts or tooling

### 4. Style Guidelines
- Keep rule logic minimal and efficient  
- Use descriptive filenames  
- Provide a brief header comment at the top of each file explaining the intent  

### 5. Pull Requests
When submitting a PR:
- Reference the issue it addresses  
- Add a short summary of your change  
- Confirm your query/script has been validated syntactically  

---

Thank you for helping strengthen **Cloud-Hun**!
